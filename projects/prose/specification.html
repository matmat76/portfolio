Analyse des besoins

La phase de spécification du projet ProSE a débuté par une analyse approfondie des besoins de l'entreprise concernant l'affichage dynamique et l'intégration des collaborateurs. Cette analyse a permis d'identifier les principales fonctionnalités à implémenter.

Besoins fonctionnels

Configuration des moniteurs

Permettre aux administrateurs de configurer à distance les paramètres d'affichage de chaque moniteur via une application Android (FortilApp).

Widgets personnalisables

Offrir un ensemble de widgets configurables pour afficher des informations pertinentes (météo, événements, anniversaires, date, disponibilité des salles, actualités, transports).

Catalogues de contenu

Permettre l'affichage séquentiel de trois types de contenus : fiches collaborateurs, fiches projets et vidéos d'entreprise.

Synchronisation des données

Assurer une communication fiable et sécurisée entre l'application Android (FortilApp), le système d'affichage (Fora) et le serveur de l'entreprise.

Besoins non fonctionnels

  • Performance : Temps de réponse rapide et gestion efficace des ressources
  • Sécurité : Communication sécurisée via Protocol Buffers (protobuf)
  • Fiabilité : Système stable avec gestion des erreurs robuste
  • Maintenabilité : Architecture modulaire suivant les principes SOLID
  • Extensibilité : Possibilité d'ajouter facilement de nouveaux types de widgets ou de contenus

Cas d'utilisation

L'analyse des besoins a permis d'identifier les principaux cas d'utilisation du système, que nous avons formalisés selon la méthode UML.

Acteurs principaux

  • Administrateur : Unique utilisateur qui configure et gère les moniteurs via l'application FortilApp
  • Serveur d'entreprise : Fournit les données à afficher (projets, collaborateurs, vidéos, etc.)
  • OS : Système d'exploitation qui gère le démarrage et l'arrêt de Fora

Cas d'utilisation principaux

Lancer Fora

Acteur principal: Administrateur

Description: L'administrateur lance Fora qui affiche l'écran configuré sur le moniteur.

Scénario principal:

  1. Fora démarre
  2. Fora affiche l'écran de chargement
  3. Fora vérifie la présence d'une configuration
  4. Fora charge la configuration
  5. Fora demande les données au serveur d'entreprise
  6. Fora affiche la présentation sur le moniteur

Configurer une borne

Acteur principal: Administrateur

Description: L'administrateur sélectionne une borne existante et modifie ses paramètres (nom, adresse IP, catalogue, volume, temps d'affichage, widgets activés).

Scénario principal:

  1. L'administrateur sélectionne une borne
  2. FortilApp charge les données de la borne sélectionnée
  3. FortilApp demande au serveur la liste des catalogues
  4. FortilApp affiche l'écran de configuration
  5. L'administrateur modifie les paramètres souhaités
  6. L'administrateur valide la configuration
  7. FortilApp sauvegarde et envoie la configuration à Fora
  8. Fora applique la configuration et affiche le contenu

Ajouter une nouvelle borne

Acteur principal: Administrateur

Description: L'administrateur ajoute une nouvelle borne qui sera ensuite affichée sur l'écran d'accueil de FortilApp avec son statut.

Scénario principal:

  1. L'administrateur ajoute une nouvelle borne
  2. FortilApp affiche l'écran de nouvelle borne
  3. L'administrateur configure les identifiants (nom et adresse IP)
  4. L'administrateur valide les identifiants
  5. FortilApp vérifie les identifiants
  6. FortilApp sauvegarde la borne
  7. FortilApp rafraîchit le statut des bornes

Rafraîchir le statut des bornes

Description: FortilApp récupère le statut de toutes les bornes enregistrées.

Scénario principal:

  1. FortilApp demande le statut d'une borne à Fora
  2. Fora répond avec son statut
  3. FortilApp enregistre le statut (Connectée, Déconnectée ou À configurer)
  4. FortilApp répète pour chaque borne
  5. FortilApp met à jour l'affichage avec les statuts actualisés

Description des cas d'utilisation principaux

Configurer un moniteur

Acteur principal: Administrateur

Description: L'administrateur sélectionne un moniteur existant dans la liste et modifie ses paramètres (nom, adresse IP, catalogue, volume, temps d'affichage, widgets activés).

Scénario principal:

  1. L'administrateur ouvre l'application FortilApp
  2. Il sélectionne un moniteur dans la liste des moniteurs disponibles
  3. Il modifie les paramètres souhaités
  4. Il valide les modifications
  5. L'application transmet la nouvelle configuration au moniteur
  6. Le moniteur applique la configuration et confirme la mise à jour

Gérer les catalogues

Acteur principal: Administrateur

Acteur secondaire: Serveur d'entreprise

Description: L'administrateur configure le catalogue à afficher sur un moniteur, déterminant ainsi les vidéos, projets et collaborateurs qui seront présentés.

Scénario principal:

  1. L'administrateur ouvre la configuration d'un moniteur
  2. Il sélectionne un catalogue dans la liste des catalogues disponibles
  3. Il définit la durée d'affichage des éléments
  4. Il valide la configuration
  5. Le moniteur télécharge le contenu du catalogue depuis le serveur
  6. Le moniteur commence à afficher séquentiellement les éléments du catalogue

Focus sur le composant Catalog

Dans la phase de spécification, nous avons défini précisément les responsabilités du composant Catalog, qui est au cœur du système d'affichage.

Fonctionnalités requises

  • Charger et stocker les données des différents types de contenu (vidéos, projets, collaborateurs)
  • Gérer les transitions entre les différents éléments à afficher
  • Respecter les durées d'affichage configurées
  • Notifier l'interface utilisateur des changements d'état
  • Communiquer avec le serveur pour obtenir les données via des requêtes HTTP
  • Gérer la mise en cache des données pour optimiser les performances
