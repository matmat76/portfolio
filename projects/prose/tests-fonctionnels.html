Tests fonctionnels: validation des exigences

Les tests fonctionnels jouent un rôle crucial dans le cycle en V, car ils permettent de valider que le système répond aux exigences définies lors de la phase de spécification. Contrairement aux tests unitaires et d'intégration qui se concentrent sur les aspects techniques, les tests fonctionnels vérifient que le système fait ce qu'il est censé faire du point de vue de l'utilisateur final.

Approche de test fonctionnel

Notre approche des tests fonctionnels s'est appuyée sur les principes suivants:

  • Tests basés sur les exigences (requirement-based testing)
  • Scénarios d'utilisation complets (end-to-end)
  • Validation du point de vue de l'utilisateur final
  • Gestion et suivi des cas de test avec Squash TM

Squash TM pour la gestion des tests

Pour gérer efficacement notre campagne de tests fonctionnels, nous avons utilisé Squash TM, un outil open-source de gestion des tests qui permet de structurer, organiser et suivre l'exécution des tests.

Avantages de Squash TM

  • Traçabilité entre exigences et tests
  • Gestion centralisée des cas de test
  • Organisation en campagnes de test
  • Suivi de l'avancement des tests
  • Génération de rapports détaillés

Cas de tests fonctionnels pour le Catalog

Pour le module Catalog, nous avons développé plusieurs cas de test fonctionnels qui vérifient que toutes les exigences sont satisfaites.

TC-CAT-001: Configuration et démarrage d'un catalogue

Exigence associée: REQ-CAT-01 - Configuration des catalogues

Pré-condition: Application démarrée, connexion au serveur établie

Étapes:

  1. Sélectionner "Configurer un catalogue" dans l'interface de gestion
  2. Entrer l'ID du catalogue "42"
  3. Définir la durée d'affichage à 10 secondes
  4. Cliquer sur "Démarrer l'affichage"

Résultat attendu: Le catalogue démarre et affiche une série de contenus

Résultat obtenu: Succès

TC-CAT-002: Séquence d'affichage et rotation des contenus

Exigence associée: REQ-CAT-02 - Affichage séquentiel

Étapes:

  1. Configurer un catalogue avec 3 vidéos, 2 projets et 4 collaborateurs
  2. Démarrer l'affichage du catalogue
  3. Observer la séquence des contenus pendant 15 minutes

Résultat attendu: Tous les types de contenu apparaissent de manière équilibrée, chaque contenu apparaît au moins une fois

Résultat obtenu: Succès

TC-CAT-003: Reprise après coupure réseau

Exigence associée: REQ-CAT-05 - Robustesse

Étapes:

  1. Démarrer l'affichage d'un catalogue
  2. Déconnecter le réseau pendant l'affichage
  3. Attendre 5 minutes
  4. Reconnecter le réseau

Résultat attendu: Le système continue d'afficher les contenus déjà chargés pendant la coupure, puis récupère les mises à jour après reconnexion

Résultat obtenu: Partiel - Délai de récupération trop long (> 2 minutes)

Campagne de tests fonctionnels

La campagne de tests fonctionnels a couvert l'ensemble des exigences du système et a permis de valider le fonctionnement global du projet ProSE.

95%

Couverture des exigences: 95%. Toutes les exigences critiques ont été validées, avec quelques ajustements mineurs nécessaires pour certaines fonctionnalités secondaires.

Résultats et adaptations

Les tests fonctionnels ont permis d'identifier certains points d'amélioration:

  • Temps de récupération après déconnexion: L'algorithme de détection et de reprise a été optimisé pour réduire le temps de récupération.
  • Équilibrage des types de contenu: L'algorithme de sélection a été affiné pour assurer une meilleure distribution des différents types.
  • Transitions entre contenus: Les transitions ont été améliorées pour apparaître plus fluides, particulièrement entre différents types de contenu.
