Architecture globale du système

La conception générale du projet ProSE repose sur une architecture modulaire et scalable, conçue pour faciliter le développement, les tests et la maintenance. Cette phase de conception établit les fondations du système et définit les interactions entre ses composants principaux.

Diagramme d'architecture globale

L'architecture se compose de trois composants principaux :

  • FortilApp (Android) : Application mobile permettant aux administrateurs de configurer et contrôler les moniteurs d'affichage.
  • Fora (C) : Système d'affichage embarqué, composé de deux modules principaux :
    • Widgets : Affichage d'informations en temps réel (météo, événements, etc.)
    • Catalog : Gestion des contenus à afficher (projets, collaborateurs, vidéos)
  • Serveur Entreprise : Stockage centralisé des données à afficher.

Principes de conception

Modulaire

Le système est conçu avec des modules indépendants et faiblement couplés, facilitant le développement parallèle et les tests isolés.

Robuste

Gestion rigoureuse des erreurs et mécanismes de récupération pour assurer la continuité du service même en cas de défaillance d'un composant.

Évolutif

Architecture conçue pour permettre l'ajout de nouvelles fonctionnalités sans modifier les composants existants.

Focus sur le module Catalog

Le module Catalog, dont j'ai eu la responsabilité, joue un rôle central dans le système d'affichage dynamique. Il est chargé de récupérer, gérer et présenter de manière séquentielle les différents types de contenu.

Responsabilités du module Catalog

  • Communication avec le serveur pour récupérer les données
  • Gestion des différents types de contenu (vidéos, projets, collaborateurs)
  • Ordonnancement de l'affichage des contenus
  • Gestion du cycle de vie des éléments affichés
  • Notification des événements au système principal

Technologies et standards

Le projet utilise les technologies et standards suivants:

  • Langage C pour le développement du système Fora
  • Protobuf pour la communication entre FortilApp et Fora
  • HTTP/JSON pour les échanges avec le serveur d'entreprise
  • Raylib pour le rendu graphique
  • UML (ISO/IEC 19505) pour la modélisation
